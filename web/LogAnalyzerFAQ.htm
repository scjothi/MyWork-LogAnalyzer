1.0 Input Related

 

How would I search for all the log statements of a specific web service?

In the ‘WEBSERVICE’  field, you can enter full or partial name of the web service you want to search. Note that this field is case sensitive and can take wild character.  For e.g , if you want to search all the logs of the web service WebSSOWS-v100,  the  search inputs can be one of: WebSSO* or WebSSOWS* or WebSSOWS-v100. Entering ‘webssows*’ would return no results since the case does not match the value in the log.

I want to search all the logs for the web service ‘VoicePortalWS-v221’. How would I do that?

Enter ‘VoicePortalWS-v221’ in the WEBSERVICE field and enter other inputs as it applies. Click ‘Get Results’.

I am not sure about the version of the web service I want to analyze. How would I search the logs?

Let’s say the name of the web service is VoicePortalWS. Enter ‘VoicePortalWS*’ in the WEBSERVICE field and enter other inputs as it applies. Click ‘Get Results’.

How would I search for a particular text in the log message?

Each field in a log event is stored as a searchable Index field. The log message itself is a separate field that contains the text or the string that is logged. In order to search the log message, you can enter search keywords that are part of the log message in one of the formats below in the ‘MESSAGE’ field. This field is NOT case sensitive.

a)      To match exactly a whole phrase, enter it surrounded by double quotes. E.G: "Unable to get database connection via JNDI" will return all the log entries that contains the quoted text.

b)      To find matches for any part of a text, enter the text without surrounded by double quotes. . E.G: JNDI facets  will return all log entries that contains either ‘JNDI’ or ‘facets’ in the log message.

c)       To match exactly a word or a term, enter it in the Message field. Note that log message entries need not be in the same case as it is in the log.

What is a search ‘term’?

A search term can be a word or a sequence of characters separated by a separator char(space, /,\ e.t.c).

How do I search logs within certain time frame?

To view the logs within a certain time frame, use the ‘FROM DATE’ and ‘TO DATE’ input fields. The date format is MM/DD/YYY. The default input is the current date starting from 12:00 AM  until 11:59 PM.  You can change the inputs to match your requirements. To change the time, select the time field(date,month,year,hour,minute) using the mouse pointer or tab key and overwrite the value.

What is the ‘INSTANCE’ field for?

This field does not apply to Production environment. It only applies to QA currently.

How do I search all the logs in a particular server?

Use the ‘SERVER’ field to filter the logs for a server from the list of servers available.

What is the long text field at the bottom of the form?

Please ignore this field.  It is just there for trouble shooting purposes.

I want to search for ‘NullPointerException’ in the log message. How would I do that?

Enter ‘java.lang.NullpointerException’ in the MESSAGE field and enter other inputs as it applies. Click ‘Get Results’.

Note that if you just enter ‘NullPointerException’ in the MESSAGE field, you won’t probably get any results unless it is present in any log message as a separate term. Normally the Exceptions are logged with their package namespace like ‘java.lang.NullpointerException’ which is considered as a single search term.

Why is there a ‘*’ by default in some text fields?

The ‘*’ is present by default in some text field as a wild character which means ‘All’. Unless you replace it with a search term the search action will not filter that field.
Do not replace ‘*’ by just empty space or nothing. It will not return any results.

2.0 Output Related

 

What does the ‘Show All’ link do?

It is meant for expanding lengthy log messages such as Exception stack trace and other long messages that are logged. Currently it appears for all log records whether they are lengthy or not.

Do the logs appear exactly as in the log file?

Yes, the log is supposed to be exactly match the log file in the server. Sometimes the format may differ, but you should be able to see exactly the message as a whole as present in the server log file.

How frequently are the logs updated in the index?

It is configurable and currently set to update every 5 minutes.

I am seeing the spinner for a long time. How should I proceed?

Try one of these:

a)      Click on the browser refresh button and retry. Note that, this will clear all the form fields.

b)      While the spinner is on, change the search criteria by narrowing the  time frame ( FROM DATE & TO DATE ) or by narrowing the search string and click ‘Get Results’

c)       Open a new window or tab and retry the search.

 I want to search a word or term in the search results. How would I do that?

Currently the only way you can search is using the browser’s ‘Find’ feature. After doing a search, press ‘ Ctrl+F’ and enter the term or the word you want to find in the search result. It will highlight all the matching terms in the current search result page.

3.0 Feature Related

 

Can I download and save the search results?

Yes you can. At the bottom of the screen there is a small strip that can be expanded. Look for the icon with ‘CSV’ on it. On clicking it, the current page will be downloaded as a Comma Separated Value(CSV) file. Each field will be in a separate column in the CSV file.

Note that the log Message field values longer than 500 characters are truncated in the CSV.

Can I save the search inputs so that I can repeat the search or email it to some body?

Currently you cannot do that. This feature will be added soon.

Can I filter search results by Thread name?

Currently you cannot do that. This feature can be added in the future if needed.

Can I filter search results by Thread name?

Currently you cannot do that. This feature can be added in the future if needed.
